# Atlas Skill

让 AI 代理在整个执行过程中保持与原始目标对齐的 prompt 技能——包括在本该漂移的长时多阶段会话中。

[English → README.md](./README.md)

---

## 问题是什么

AI 代理会悄悄漂移。它缩小任务范围、把后端打桩、弱化测试、改了被要求保留的文件,然后汇报"成功"。代理不会感觉这是背叛——它感觉像是在推进进度、修 bug、做了一个无害的简化。

靠问模型"你诚实吗?"解决不了这个问题。那个问题也交给了同一个已经漂移了的判断器。

## Atlas Skill 做什么

在执行前,它把请求冻结成一份**目标合同**:必须做什么、不能做什么、必须保留什么、范围边界,以及每一项如何验证。

在执行过程中,它通过**阶段检查点**来治理进度——在范围变更、悄悄缩小、用 mock 替代、验证失败、或"无证据地声称无影响"之前停下来。每次停止都有**事件编号**记录——目标变更、偏离、未验证项不能被埋进进度摘要里。

在标记完成之前,它跑一次**对抗性审计**:不是自我总结,而是对每个合同项的结构化检查,要求提供具体证据。模型必须**假设自己已经漂移**,主动找它少做了什么。

## 为什么它能撑过长对话

上下文压缩会丢掉约束。在任何压缩或会话接力之后,Atlas 在继续工作前会重新发布确认过的合同**以及**一份五条规则锚点。这就是为什么它能在一个对话里维持 70 多个阶段性任务而不跑偏。

## 怎么使用

把 [`SKILL.md`](./SKILL.md) 的内容复制到以下任一位置:

**Codex / GitHub Copilot 代理** — 粘贴到代理的系统指令里,或作为工作区里的 `.md` 技能文件。

**Claude 项目** — 粘贴到项目的"Project Instructions"里。

**Cursor 代理 / 系统提示** — 粘贴到自定义系统提示或 `.cursorrules` 的顶部。

**任何遵循系统提示的模型** — 在哪工作,就在哪粘贴。

无需安装,无需依赖。直接用。

## 你会得到什么

当 Atlas 激活(Gate Mode),代理在动任何代码前先输出一份目标合同:

```
Atlas 事件：
- 事件编号：A1
- 类型：目标合同
- 触发来源：Skill 主动触发
- 阶段：P0
- 停止状态：停止

Atlas 目标合同

目标：
- 实现完整的 JWT 用户认证,包含登录、注册、token 刷新和受保护路由中间件。

必须做：
- [M1] 真实 JWT 实现——不得使用硬编码 token 或 mock 认证（硬性,
        来源："完整实现",验证：登录返回签名 JWT,中间件拒绝无效 token）
- [M2] 持久化会话——token 在服务器重启后仍有效（硬性,
        来源："持久化",验证：重启后 token 仍可用）

禁止做：
- [N1] 任何层面不得使用 mock/stub/占位符认证（硬性）
- [N2] 不得修改现有用户表结构（硬性）

必须保留：
- [P1] 现有 /api/users 路由及响应结构（硬性）

完成检查：
- [C1] 合法凭据登录 → 返回 JWT
- [C2] 过期 token 请求 → 401
- [C3] 无 token 访问受保护路由 → 401

ATLAS_STOP: 等待用户确认后再继续。
```

确认后,复杂任务会建立**阶段账本**,在每个阶段边界跑**阶段检查**。声明完成之前跑对抗性审计——对每个合同项检查实际 diff 和测试输出,不靠记忆。

---

## 它阻止什么

| 悄悄失败的方式 | Atlas 的应对 |
|---|---|
| 把后端 mock 掉然后叫完成 | 禁止做 → 动作前硬性拦截 |
| 弱化或删掉不过的测试 | §5 停止动作 → 偏离通知 |
| 把"完整实现"缩小成只有前端 | 禁止范围降级 → 必须披露 |
| 上下文压缩后忘掉保留约束 | 压缩后重发合同 + 规则锚点 |
| 没检查就声明阶段完成 | 阶段检查要求每个合同项有证据 |
| "我确信这个改动是隔离的" | 禁止自我裁定影响 → 证明或标记未验证 |
| Final Audit 说完成但有 mock | 对抗性清单：必须检查运行时代码路径 |

## 兼容性

适用于所有遵循系统提示的模型:

- **GitHub Copilot / Codex** — 主要开发环境
- **Claude**（项目指令、系统提示）
- **Cursor**（代理模式、`.cursorrules`）
- **ChatGPT**（自定义指令、系统提示）
- **其他任何遵循指令的代理**

Skill 自动检测用户当前语言并本地化所有输出标签。中英文已充分测试,其他语言遵循同样的本地化规则。
