# Atlas

一个 prompt 技能系列,核心只做一件事:**让有能力的 AI 代理对目标诚实**——让一个强模型始终对你真正要的东西负责,而不是在执行过程中悄悄把它换成一个更省事、或"看起来更合理"的版本。

[English → README.md](./README.md) · [示例 → EXAMPLES.md](./EXAMPLES.md)

---

## 谁该用它

Atlas **不是**给弱模型擦屁股的,也**不是**为了提升代码质量。一个强模型配上一份像样的 `CLAUDE.md`,本来就能写出好代码——而且模型只会越来越强,这个下限只会越来越高。

强模型**解决不了、甚至会让它更严重**的,是另一件事:在一个任务的深处,有能力的代理会替你做一连串决定。它擅自替换一个方案、悄悄收窄一个范围、自己挑一个默认值、"顺手优化"了旁边的东西。模型越聪明,这些决定看起来**越合理**,于是你越容易不细看就放过去——而当其中某一个悄悄偏离了你要的东西时,你越难发现。最后你其实是在赌:它每一次、在每个你根本没看见的决定上,都恰好选对了。

Atlas 不让代理变聪明。它让这些决定、捷径、和未经验证的声称**无法被藏起来**——这样你才能在真正会发生这种"悄悄漂移"的长任务、高风险任务上信任一个强模型。去看 **[EXAMPLES.md](./EXAMPLES.md)**:那是一次真实的对照,两个版本都写出了好代码,但只有一个让用户参与了那个"改动了原始需求"的决定。

> 它换了种方式,但没经过同意。如果是好的方式,确实没问题——但它真能在每种情况下都自主选对吗?其实就是个概率,去赌它自己选对。第二个让你自己选,同时还给出了它自己的打算。
>
> It swapped in another approach without asking. This time it was a good one — but can it really pick right in *every* case? That's just probability; you're betting it chooses correctly on its own. The other one lets you choose — while still telling you what it intended to do.

## 问题是什么

一次悄悄的目标改动,从内部看很少像背叛。它像在推进——像在修构建、像一个无害的简化、像一个合理的替换。所以靠问模型"你诚实吗"没用——那个问题交给的是同一个已经漂移的判断器。

这个问题没法靠把模型变聪明来解决,只能靠改变它周围的**流程**:一开始就冻结目标,把会改动目标的决定浮出来,并让每一次偏离都变成一个你看得见、能批准的可审计事件。

## 两个技能

| 技能 | 角色 | 何时运行 |
|---|---|---|
| **[atlas-contract](./atlas-contract)** | 对话内治理。把请求冻结成可审计的**目标合同**,按任务复杂度调整介入程度(轻 / 中 / 重),在会改动目标的**动作**(替换、收窄、mock、删除、弱化测试)处停下——而不是让模型自己判断风险,并在声明完成前跑一次对抗式**最终审计**,任何没被证明的项一律标 `Unverified`,绝不写"完成"。 | 每个非平凡的编码任务。 |
| **[atlas-ledger](./atlas-ledger)** | 跨会话记忆(复利)。当一次漂移被**抓到**,把教训提炼成可复用的项目条款(`WHEN / DON'T / INSTEAD`),经你确认后写入 `Atlas.md`。 | 只在漂移被抓到之后。 |

**它们怎么形成复利:** atlas-contract 在当下守住目标;atlas-ledger 把每次抓到的漂移变成 `Atlas.md` 里的永久条款。下次 atlas-contract 建合同时,会读取相关条款、预填成护栏——于是项目的防御线随着每次抓取越来越厚,而不是一错再错。

```
atlas-contract  ──抓到漂移──▶  atlas-ledger  ──写入条款──▶  Atlas.md
       ▲                                                        │
       └──────────── 下次建合同时载入条款 ◀──────────────────────┘
```

## 它长什么样

在一个非平凡的任务上,代理会在动代码之前冻结目标、停下来等你确认——这样任何会改动你需求的决定,都在它被执行**之前**浮出来,而不是之后:

```
Atlas Goal Contract

Goal: 给每次转换加一条实时新闻标题,从真实来源获取。

Must Do:
- [M1] 在请求时获取一条真实标题 (hard)

Must Not Do:
- [N1] 不得使用写死、mock、伪造或占位的标题 (hard)

Preserve:
- [P1] 现有的转换 / 汇率 / 历史 / 删除功能必须保持正常 (hard)

Summary: 我会按货币获取真实标题并接入,不破坏任何现有行为。

ATLAS_STOP: 等待你确认后再开始。
```

完整的前后对照见 **[EXAMPLES.md](./EXAMPLES.md)**。

## 怎么使用

每个技能就是一个 `SKILL.md`。把内容复制到:

- **Codex / GitHub Copilot 代理** — 系统指令,或工作区里的 `.md` 技能文件
- **Claude** — 项目指令(Project Instructions)
- **Cursor** — 系统提示或 `.cursorrules`
- **任何遵循系统提示的代理** — 它读系统提示的地方

无需安装、无依赖。从 `atlas-contract` 开始;想让项目记住自己犯过的错,再加上 `atlas-ledger`。

## 诚实的边界

这些技能是由它们所约束的同一个模型来执行的。它们抬高了目标保真的底线、让悄悄漂移在结构上更难,但一个足够漂移的模型仍可能在没做完的工作上产出一份看起来干净的审计——因为那道对抗式审查也是它自己跑的。对高风险或长时间运行的工作,真正的外部兜底是**代码层的机械闸**——在工具动作执行前就拿它和合同比对、不让模型自己判断。那一层是 **[atlas-agent](https://github.com/wede-wx/atlas-agent)**。把 Atlas 技能当作必要的一层,而不是完整方案。
