# Atlas

一个 prompt 技能系列,核心只做一件事:**让 AI 代理对目标诚实**——阻止它悄悄缩小、mock、隐藏、或假装完成,并让任何偏离都无法被埋掉。

[English → README.md](./README.md)

---

## 问题是什么

AI 代理会漂移。给它一个真实任务,它会悄悄缩小目标、把后端 mock 掉、弱化测试、改掉被要求保留的文件,然后汇报"完成了"。它不觉得这是背叛,它觉得这是在推进。靠问模型"你诚实吗"没用——那个问题交给的是同一个已经漂移的判断器。

这个问题没法靠把模型变聪明来解决,只能靠改变它周围的**流程**。

## 两个技能

| 技能 | 角色 | 何时运行 |
|---|---|---|
| **[atlas-contract](./atlas-contract)** | 对话内治理。把请求冻结成可审计的**目标合同**,按任务复杂度调整介入程度(轻 / 中 / 重),在破坏性或改变范围的动作前停下,并在声明完成前跑一次对抗式**最终审计**。 | 每个非平凡的编码任务。 |
| **[atlas-ledger](./atlas-ledger)** | 跨会话记忆(复利)。当一次漂移被**抓到**,把教训提炼成可复用的项目条款(`WHEN / DON'T / INSTEAD`),经你确认后写入 `Atlas.md`。 | 只在漂移被抓到之后。 |

**它们怎么形成复利:** atlas-contract 在当下守住目标;atlas-ledger 把每次抓到的漂移变成 `Atlas.md` 里的永久条款。下次 atlas-contract 建合同时,会读取相关条款、预填成护栏——于是项目的防御线随着每次抓取越来越厚,而不是一错再错。

```
atlas-contract  ──抓到漂移──▶  atlas-ledger  ──写入条款──▶  Atlas.md
       ▲                                                        │
       └──────────── 下次建合同时载入条款 ◀──────────────────────┘
```

## 怎么使用

每个技能就是一个 `SKILL.md`。把内容复制到:

- **Codex / GitHub Copilot 代理** — 系统指令,或工作区里的 `.md` 技能文件
- **Claude** — 项目指令(Project Instructions)
- **Cursor** — 系统提示或 `.cursorrules`
- **任何遵循系统提示的代理** — 它读系统提示的地方

无需安装、无依赖。从 `atlas-contract` 开始;想让项目记住自己犯过的错,再加上 `atlas-ledger`。

## 目录结构

```
atlas-contract/SKILL.md    对话内目标治理(从这里开始)
atlas-ledger/SKILL.md      被抓到漂移的复利记忆 → Atlas.md
LICENSE                    MIT
```

## 诚实的边界

这些技能是由它们所约束的同一个模型来执行的。它们抬高了目标保真的底线、让悄悄漂移在结构上更难,但一个足够漂移的模型仍可能在没做完的工作上产出一份看起来干净的审计。对高风险或长时间运行的工作,真正的外部兜底是**代码层的机械闸**——在工具动作执行前就拿它和合同比对、不让模型自己判断。那一层是 **[atlas-agent](https://github.com/wede-wx/atlas-agent)**。把 Atlas 技能当作必要的一层,而不是完整方案。
